# Financial Engineering Assignment
## Subject: Computational Financial Modeling (SQL)
### 1. Assessment Objectives

The objective of this assessment is to apply data science principles to financial mathematics using SQL.
Students must demonstrate the ability to translate mathematical models into precise computational
queries. Key focus areas include:
• Algorithmic Implementation: Developing queries that accurately model financial data.
• Mathematical Translation: Mapping complex formulas into optimized SQL logic.
• Numerical Precision: Managing DECIMAL data types to ensure high-fidelity financial reporting (ZAR).
• Model Validation: Verifying query accuracy against known reference points.
### 2. Technical Requirements
• Environment: Microsoft SQL Server Management Studio (SSMS).
• Database Schema: Use FinancialEngineeringDB and the table FinancialTasks.
• Data Consistency: All financial outputs must be formatted to exactly two decimal places.
• Validation: Results must align exactly with the provided Reference Outputs.
